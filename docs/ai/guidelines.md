Here's a distilled best practice workflow for coding with AI:

**Required Tools:**

- Large context reasoning model (e.g., GPT-4 or Gemini)
- Code-focused AI assistant (e.g., Cline with Claude)
- Code base consolidation tool

**Workflow Steps:**

1. **Planning Phase**

- Collaborate with reasoning model to define tech stack and app structure
- Get detailed, phased implementation plan

1. **Setup & Documentation**

- Create `IMPLEMENTATION.md` with the plan
- Create `PROGRESS.md` to track status
- Set up initial project structure

1. **Iterative Development Loop**

- Have AI assistant implement one phase at a time
- Test functionality after each phase
- Update progress tracking
- Start new chat for each phase
- Maintain consistent progress documentation

1. **Maintenance & Refactoring**

- Monitor file lengths (keep under 300 lines)
- Use reasoning model for refactoring plans
- Regularly consolidate and review full codebase
- Return to reasoning model for major architectural decisions

**Key Principles:**

- Break work into discrete phases
- Maintain clear documentation
- Use reasoning model for high-level decisions
- Use code assistant for implementation
- Test frequently
- Keep code organized and manageable

This workflow scales well while preventing technical debt and maintaining code quality.
